## 1. North-star decisions (locked)

| # | Decision | Locked answer |
|---|---|---|
| D1 | Report failure mode when inputs are incomplete | **Structured gaps + partial report.** Reports always render. Affected fields show `DATA_UNAVAILABLE`. A top-level `gaps: DataGap[]` manifest lists every missing input with `field`, `reason`, `severity` (`CRITICAL`/`WARNING`), `action`. **Never** silent zeros, **never** hard 422s. |
| D2 | Idempotency key for `Institution` | Composite `(workspaceId, seedKey)` where `seedKey` is a new nullable column populated only by the seed pipeline. Real-world institution creation is unaffected. |
| D3 | Action registry shape | Thin layer. `ActionMeta` interface (id, label, module, permissions, requiresConfirm, audit, idempotencyKey) + `register()`/`dispatch()` methods. Registry collects at boot via per-module bootstrap classes (e.g. `AlmActionsBootstrap`), dispatches via `POST /api/actions/:id/dispatch`, audits to existing `audit_logs` table. |
| D4 | ReportPreflight failure semantics | `check()` never throws. Returns `PreflightResult` with `ready: boolean` + unified `gaps[]`. Callers branch on `ready`. **No `assertReady()`** that raises — would re-introduce hard-fail and conflict with D1. (2026-04-07) |
| D5 | Action registry granularity | **Thin.** Decorator/metadata + dispatch + audit. No middleware, no orchestration, no workflow engine. Each action wraps an existing service method. (2026-04-07) |
| D6 | Audit log storage | Existing `audit_logs` Prisma table (`prisma/schema.prisma:1400`). No new schema. Action id → `action`, "action_dispatch" → `resource`, input/result → `changes`, timing/gaps → `metadata`. Action dispatches live alongside login/upload/payment audit. (2026-04-07) |
| D7 | Golden test failure mode | Snapshot file at `test/golden/*.expected.json`. **Drift fails CI.** No auto-update on assertion failure. To regenerate, set `UPDATE_GOLDEN=1` and review the diff in PR. The manual update IS the gate. (2026-04-07) |

## 4. Conventions (FAANG-bar, project-specific)

- **Bloomberg density UI**, no card grids. Use `MetricStrip`, `DataRow`, `DataTable`. (Carried from user-level feedback.)
- **Trust the presenter** — services return raw data; presenters format. Don't double-format.
- **No silent fallbacks.** Every catch must either surface a typed error or emit a `DataGap`. Logging is not a substitute for either.
- **Idempotency first.** Anything that can run twice should produce the same result the second time, or explicitly say what changed.
- **Migrations are forward-only.** Never edit a migration that's been applied. Generate a new one.
- **Pin snapshots.** Any read used in a report should come from a snapshot ID, not "now". Tests must control time.

### 2026-04-16 Agent Runtime Hardening Pass

Three P0 defects surfaced during Wave-03 code read and fixed with regression guards. All details in §5's 2026-04-16 "Agent runtime FAANG hardening" landing. Key takeaway for the next session:

- **Risk Monitor crons are now real.** Before this pass, `@Cron` decorators on `AgentSchedulerService` were decorative — `ScheduleModule.forRoot()` was missing from `AppModule`. In any deploy after this commit, the daily/weekly/monthly cadences will actually fire. Confirm by grepping boot logs for `ScheduleExplorer` or triggering `curl /api/agent-runs/schedule` and observing a dispatched run within the cron window.
- **The `await X?.() ?? Y` pattern is an anti-pattern in this codebase.** If you see it anywhere else (grep confirmed `agent-alert-notifier` was the only offender), replace with an explicit `typeof === 'function'` branch. A void-returning RHS will cause double-execution of the fallback.
- **Env schema is now the source of truth.** If you add a new `process.env.FOO` read anywhere in `backend-node/src`, add a matching Zod entry to `env.schema.ts` in the same commit — otherwise typos in Railway pass startup silently.
- **Shared-flag rule for anything read from two sites.** Any env var (or other runtime flag) consumed from more than one place gets a single helper with a test-locked truth table. `isSchedulerDisabled()` at `src/agents/scheduler/scheduler-flag.util.ts` is the template. The enemy is two sites each interpreting the same string with different rules (JS truthiness vs `=== 'true'`), which produces UI-vs-reality lies that only surface during incidents.
- **Operator-facing env var names must round-trip.** If `.env.example` documents a var, the code must honor it. `AgentCostCircuitBreakerService.resolveCapCents` now resolves both `LLM_COST_CAP_USD_CENTS` (precise, legacy) and `LLM_COST_ALERT_THRESHOLD_USD` (documented) with cents-form priority and fractional-USD rounding. The pattern to copy: static resolver taking `env: NodeJS.ProcessEnv` so the spec can table-test it without constructing the service.
- **`parseInt` is banned for env parsing.** `parseInt('100abc', 10)` returns `100` and `parseInt('10.5', 10)` returns `10` — both silent coercions hide operator mistakes. Prefer `Number(raw)` plus explicit `Number.isFinite` / `Number.isInteger` / range checks, **or** let the Zod schema coerce with `z.coerce.number().int().min(...)` and trust the boundary.
- **Every long-running runtime operation needs a deadline.** `AgentRunnerService` now enforces `AGENT_RUN_TIMEOUT_MS` via a run-scoped `AbortController` whose signal threads through to each `tools.invoke` call. If you add a new runtime that can block on external I/O (LLM, DB, HTTP), give it the same pattern: one `AbortController` per operation, `setTimeout(...).unref()`, pass `signal` to every downstream call, clear in `finally`, translate abort into a distinct terminal status (here `TIMED_OUT` vs `FAILED`). Infrastructure that exists but never fires (like `AgentRunService.timedOut` did before this pass) is worse than no infrastructure — it lies in code review.
- **Every env var must round-trip end-to-end.** Schema-validation is necessary but not sufficient. `MAX_AGENT_TOKENS` and `ANTHROPIC_BETA_HEADER` both passed schema validation for weeks while being totally ignored by the code that was supposed to consume them. The new rule: when adding an env var to `env.schema.ts`, grep for its reads *in the same commit* and either wire the consumer or add a TODO with the consumer's filename. If you see a schema entry with no matching read in `git grep`, treat it as a latent bug.
- **Money math never touches floating-point USD.** Compute in integer cents from the start. `tokens × rate / 10_000` stays exact; `(tokens × rate / 1_000_000) × 100` drifts on any non-representable binary fraction (the 22.5-cent case is ubiquitous because it's the smallest denomination that triggers it). The `LlmBridgeService.computeCostUsdCents` implementation is the reference pattern for anything that converts a token/unit count to cents.
