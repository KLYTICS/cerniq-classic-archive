# CERNIQ Value Proposition

## Repo-Grounded Proposition

CERNIQ provides a secure upload-to-report ALM workflow for cooperativas and credit unions, combining balance-sheet ingestion, institutional risk analytics, and bilingual PDF delivery in one system.

## Confirmed From Code

- secure or authenticated institution/report flows exist
- ALM analytics exist
- bilingual reporting exists
- paid report workflow exists
- portal retrieval exists

Evidence:

- `backend-node/src/auth/*`
- `backend-node/src/alm/*`
- `backend-node/src/billing/*`
- `backend-node/src/portal/*`
- `frontend/app/page.tsx`
- `frontend/app/portal/*`

## Inferred Advantage

- CERNIQ can win more on clarity, speed, bilingual delivery, and implementation simplicity than on claiming the deepest enterprise model stack today.
- The strongest near-term differentiator is not “financial intelligence OS.” It is “institution-ready ALM reporting without the usual manual overhead.”

## Claims To Avoid Right Now

- public financial intelligence platform
- Bloomberg-style infrastructure
- enterprise risk operating system
- Rust-first institutional compute core

Those claims are directionally compatible with the roadmap, but they are not yet supported by the current implementation.

## Recommended Messaging Layers

### Internal

CERNIQ is an enterprise ALM analytics project for design partners, with a path toward broader institutional risk infrastructure once model governance and data infrastructure mature.

### External, Quiet Design-Partner Motion

CERNIQ is an experimental ALM analytics engine for cooperativas and credit unions, built with early institutional partners to streamline data ingestion, analysis, and bilingual reporting.

### Public, If Needed

CERNIQ helps institutions generate bilingual ALM reports through secure balance-sheet upload, risk analysis, and board-ready PDF delivery.
