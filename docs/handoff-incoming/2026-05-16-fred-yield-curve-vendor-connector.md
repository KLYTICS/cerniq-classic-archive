- 2026-05-16 — **feat(market-data): FRED vendor connector + US Treasury yield curve surface + dedicated /alm/market-rates page.** First connector landed under the explicit "vendor connectivity to all financials within the space" initiative — a load-bearing scaffold that proves the existing `MarketDataService` orchestrator extends cleanly to new providers while keeping KLYTICS discipline (circuit breaker, telemetry, DataGap-on-miss, lineage) intact. **Why FRED first:** the US Treasury Constant-Maturity yield curve is the canonical risk-free reference every ALM model implicitly stress-tests against (duration, EVE, NII shocks) and the existing service surface had ZERO interest-rate / macro coverage — only securities pricing via Yahoo + crypto via CoinGecko. FRED is free, public, and quoted by the St. Louis Fed; no contract negotiation required. **Backend (~700 LOC):** (1) `FredProvider` (~260 LOC) — fetch-based HTTP client targeting `https://api.stlouisfed.org/fred/series/observations`; reads `FRED_API_KEY` env at call-time so test setups can mutate; degrades gracefully when key is missing (logs warn, returns null, no throw); parses FRED's `"value": "."` missing-sentinel correctly (Rule 1 — never silent-zero on a missing observation). Public methods: `getLatestObservation(seriesId)`, `getYieldCurve()` (composes 11 US Treasury CMT tenors via `Promise.allSettled` so partial-tenor outages still produce a curve), `getInterestRate(seriesId)`, `getEconomicIndicator(seriesId, opts)`, `getFXRate(seriesId, base, quote)`. **Yield-curve inversion detection** built in: 10Y-minus-2Y spread; flagged with `inverted: boolean` + `invertedDetail: "10Y−2Y spread = -0.18%"`. (2) New DTOs in `dto/macro.dto.ts` — `YieldCurveDto` / `YieldCurvePointDto` / `InterestRateDto` / `EconomicIndicatorDto` / `FXRateDto` — all carry `provider: 'fred'` + `seriesId` + `asOf` for audit lineage (KLYTICS Rule 7). (3) `MarketDataService` extended — `getYieldCurve()`, `getInterestRate()`, `getEconomicIndicator()`, `getFXRate()` all routed through the existing `fetchWithTelemetry` so the provider-health tracker + circuit breaker + provider-failure log apply uniformly with the securities-pricing path; per-method caches with sensible TTLs (1h for rates, 6h for slow indicators like CPI/GDP). Stale-cache fallback on yield-curve: if upstream fails AND cache is <24h, serves cached curve with a warn-log; otherwise returns null so UI surfaces DataGap. (4) Two new public endpoints — `GET /api/market-data/yield-curve` + `GET /api/market-data/interest-rate/:seriesId`. SeriesId validated with a strict alphanumeric+underscore regex; failure returns HTTP 503 with a structured DataGap body (provider name + remediation action) — UI renders that explicitly per Rule 1. (5) `FredProvider` wired into `MarketDataModule` providers list. **Frontend (~250 LOC):** (1) `useYieldCurve` hook (`frontend/lib/hooks/use-yield-curve.ts`) — discriminated-union return shape `{state: 'idle' | 'loading' | 'data' | 'data-gap' | 'error'}` so the UI MUST handle the data-gap branch — there is no fallthrough-to-zero render path. Aborts via `AbortController` on unmount; honors backend's `__dataGap: true` payload as a first-class state. (2) `YieldCurveCard` component (`frontend/components/risk/YieldCurveCard.tsx`) — Bloomberg-density rendering with recharts LineChart + a per-tenor data table on the side; inversion banner flags when the recession-signal condition holds; loading + error + data-gap states each have their own visually distinct treatment so the operator can't confuse them. (3) New page at `/alm/market-rates` (`frontend/app/alm/market-rates/page.tsx`) — dedicated home for live macro reference data. Deliberately distinct from the existing `/alm/yield-curve` page (which fits a Nelson-Siegel curve to an institution-specific portfolio) — these are different concerns: observed market data vs. modeled institution curve.
