# CERNIQ Product Discipline Prompt

> This prevents feature creep. Every agent and engineer must follow this.

## Rule

If a feature does not improve the **ALM upload-to-report workflow**, it should not be added to the product.

## The Gate

Before building any feature, ask:

1. Does this help a cooperativa CFO upload a balance sheet faster?
2. Does this improve the accuracy of the ALM analysis?
3. Does this make the bilingual report more useful to a board?
4. Does this reduce the time from upload to download?

If the answer to all four is **no**, the feature is deferred.

## Expansion Sequence

CERNIQ must win the ALM reporting wedge **before** expanding into:

```
Phase 1 (NOW):  ALM Reporting          ← win this first
Phase 2:        Liquidity Risk Analytics
Phase 3:        Portfolio Risk
Phase 4:        Treasury Management
Phase 5:        Regulatory Intelligence
```

Each phase unlocks only after the previous one has **paying customers and proven retention**.

## What "Winning the Wedge" Means

- ≥ 5 paying institutions
- ≥ 3 months of repeat report generation
- Net Promoter Score > 40
- CFO references available for case studies

Until these criteria are met, no new product surface should be started.
